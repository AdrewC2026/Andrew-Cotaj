05

Multithreaded HFT Trading System

Architected a multithreaded engine with a select()-driven TCP order-entry loop, a multicast market-data worker, and a ~2 ms strategy loop, synchronized with fine-grained locks. Implemented the full exchange wire protocol from scratch (little-endian binary LOGIN/NEW/MODIFY/DELETE + ACK/REJECT/FILL/CLOSE) with a client-side order-lifecycle state machine. Built a two-layer risk system — generic risk gate (rate limiting, price bands, exposure caps) and a competition rule controller enforcing hard position limits and a PnL floor that no strategy path can bypass. Solved the order-book cold-start via snapshot-replay → live handoff by sequence matching, modeled ETF creation/redemption against a REST service, and validated everything with 8 pytest modules using deterministic fakes.

Python Multithreading TCP / UDP multicast Risk systems pytest

Notre Dame HFT Technologies  ·  Research

06

S&P 500 ETF Arbitrage & iNAV Research

End-to-end Python pipeline reconstructing intraday ETF fair value (iNAV) from ~500 underlying constituents' NBBO quotes plus a cash/fund-flow term, across five ETFs (SPY, IVV, VOO, and ±3× SPXL/SPXS) and 50M+ WRDS/TAQ ticks per session in columnar Parquet. Implemented size-weighted micro-price and a daily-reset leveraged-iNAV model; quantified mispricing in basis points, AR(1) mean-reversion half-lives (≈60–290 ms), and lead-lag dynamics. Built a latency-and-execution simulator across co-located (~22 µs) → remote (~1.6 ms) tiers with fees + slippage, and ran event studies (FOMC, April 2025 tariff shock). Key result: no episode cleared the magnitude + duration bar to be net-profitable after costs — empirically demonstrating S&P 500 ETFs are efficiently arbitraged at sub-millisecond timescales.

Python · pandas · NumPy WRDS / TAQ · NBBO PyArrow / Parquet Market microstructure Latency modeling
