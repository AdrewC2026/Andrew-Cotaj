07

OxCaml Matching Engine — Compile-Time Invariants, Mode-Safe Execution

Exchange-side limit-order-book engine built around a pure state transition, apply : book → command → book × events.

  • → A GADT/phantom-typed order lifecycle and abstract types (behind .mli signatures) make a resting market order or a negative price uncompilable.
  • → In OxCaml — Jane Street's compiler — the matching hot path is allocation-free via the local mode (0 bytes/match, no GC tail).
  • → Multi-domain order gateway is data-race-free by construction via portability/contention modes — a shared mutable book is a compile error.
  • → Correctness: a qcheck suite (share conservation, no crossed book, FIFO, price-time priority, deterministic replay). Green in CI against the OxCaml compiler.
OCaml · OxCaml · Dune GADTs · phantom types local · portability modes qcheck (property testing)
